# Converloop Design

Converloop 是一个本地优先的 AI 语言学习桌面端。v1 的核心链路已经完成:用户在真实对话中练习目标语言,系统同时给出自然回复、结构化批改、长期记忆和后续训练。

本文记录当前产品形态、核心设计原则和后续开发指导。字段、prompt、migration、provider 适配和具体实现以代码与测试为准。

## 当前产品形态

**对话工作台。** 主流程是目标语言聊天。Conversation Agent 负责自然流式回复,Tutor Agent 并行批改,Coach Panel 展示反馈、系统记下的学习信号、到期复习项和自定义观察结果。多会话、置顶、日期分组、滚动摘要、上下文用量、`/btw` 离档问答、会话分支和场景调整都已落地。

**学习记忆。** SQLite 保存结构化掌握项和证据时间线;Markdown 学习者档案保存个人事实、兴趣、长期偏好、正在练习的内容和用户手写 notes。代码负责计数、状态迁移、复习候选选择和写入校验;LLM 负责观察、解释、整理和提出建议。

**批改与表达缺口。** Tutor Agent 输出完整改正版、更自然说法、错误 span、母语解释、严重程度和可追踪 key。母语 / 混说输入会进入表达缺口链路,UI 教用户“这个意思怎么说”,并让该缺口后续能在对话、专项课或训练里再次被引出。

**复习系统。** 复习不是单独刷卡,而是由代码选出 due review、薄弱项和已掌握脚手架,再自然进入普通对话、专项课、弱项闪练、听写和情景演练。正确使用、错误、曝光和表达缺口都保留证据。

**训练中心与专项课。** 已有情景演练、听写、弱项闪练、自定义 `converloop/drill@1` 训练文档、训练观察者和节末训练报告。专项课是老师型会话,可围绕今日复盘、语法、表达缺口或用户目标集中练习。Task Agent 可把开放式学习目标生成学习项目和专项课草案。

**能力库。** 内置能力和自定义 Agent 按入口展示:对话、批改、课程、训练观察者、回复讲解、双语对照、划词解析、会话动作等。用户可以启停、隐藏、追加补充指令,也可以创建 observer、action、reply transformer,并通过 package 导入 / 导出技能和课程。

**语音与阅读。** 支持 Soniox 实时 STT、OpenAI 兼容批量 STT、本地 Parakeet、本地 Qwen3-ASR、Edge Read Aloud、MiMo TTS、按需讲解、双语阅读、划词解析和“磨耳朵”听力页。

**本地数据。** 首启引导、provider 设置、加密密钥存储、可读 JSON 备份、设置镜像、证据预览、学习数据自然语言编辑和中英文界面都已实现。

## 核心设计原则

**真实对话优先。** 学习发生在用户想表达真实意思的时候。回复先接住用户意图,批改和记忆在旁边支持主流程。

**代码拥有长期记忆。** LLM 给离散观察和解释,代码负责计数、状态、持久化、候选选择和写入安全。任何影响长期学习数据的动作都应可审计、可测试,或经过用户确认。

**两层记忆各司其职。** SQLite 是可查询、可排序、可重算的事实层;Markdown 档案是可读、可编辑、可喂给对话 agent 的叙述层。新增功能应先判断需要哪一层,很多功能会同时读取两层但只由代码写结构化层。

**复习进入使用场景。** 新能力应优先让用户在对话、任务、专项课或训练中主动产出,而不是把复习变成孤立清单。

**能力可组合但边界清楚。** Conversation、Tutor、Profile Maintainer、Learning、Drill、Observer、Action、Transformer 各自承担一个清晰入口。新增能力时先选入口,再选读取范围和写入方式。

**本地可控。** 学习记录、档案和设置应能导出、恢复和解释。密钥与令牌走本地加密存储,不进入备份。

## 后续开发指导

### 选择正确入口

- 普通聊天体验变化:优先改 Conversation Agent 或会话修饰符。
- 批改、表达缺口、正确证据:优先改 Tutor Agent 和代码侧记账测试。
- 长期画像、个人事实、偏好整理:优先改 Profile Maintainer 或档案 UI。
- 有明确教学目标的一段练习:做专项课。
- 有固定互动机制的训练:做 drill 文档和运行时枚举。
- 每轮附加观察:做 observer。
- 从当前对话派生新会话或课程:做 action。
- 回复按钮、划词、改写、双语、解释:做 transformer。

### 改学习数据

- `mastery_key` 稳定性优先于 label 漂亮程度。
- 任何自动写入长期记忆的逻辑都要保留证据来源。
- LLM 产出的写入建议先进 preview / proposal,由代码校验后执行。
- 调整记账公式时同时补测试,并确认旧事件仍可重新解释。

### 改 agent

- prompt 变更以实现文件和测试为准,文档只记录原则变化。
- 让 agent 读最小必要上下文;宽上下文适合摘要,窄上下文适合稳定执行。
- 热路径保持轻:普通对话里的首 token 延迟优先级高。
- 新 agent 需要说明入口、触发时机、读取范围、输出位置和写入权限。

### 改训练与专项课

- 训练机制由代码枚举控制,prompt 只描述教学任务。
- 听写、弱项闪练这类模式要明确是否写 mastery,以及写入哪个维度。
- 专项课默认像老师一样直接反馈,需要进入长期记忆时走课程回顾或用户确认。
- 自定义训练和 package 导入要继续保留权限预览和安全白名单。

### 改 UI

- 第一屏优先可练习,设置和说明服务主流程。
- 中心区保持轻,批改、记忆、观察和工具放到 Coach、折叠区或渐进披露中。
- 新功能要覆盖空态、加载态、错误态、窄屏和键盘焦点。
- 文案尽量说用户任务,少暴露内部 hook、schema、job 等实现词。

### 改平台与数据

- macOS 和 Windows 都是一等目标。原生能力集中封装,平台差异走薄 port 和 CSS 平台标记。
- 备份里只放用户数据和非密设置。
- 设置类用户资产要进入设置镜像,纯 UI 痕迹保持轻量。
- provider、STT、TTS 新增适配时复用现有密钥和错误显示模式。

## 文档规则

`README.md` 面向 GitHub 读者,介绍项目和已实现能力。`docs/design.md` 面向开发者,记录当前产品形态和后续开发判断标准。具体实现细节放在代码、类型、测试和必要注释里。
